## Section 3: How It Works

### Section Headline
```
The proactive loop, in 3 steps.
```

### Visual: Architecture Diagram
```
[Camera] → [Perception Model] → [Memory Store]
                                       ↓
                                 [Reasoning Loop]
                                       ↓
                                 [Display + Audio] → [Whisper to user]
```

### Section Body
```
1. Perceive: 30 FPS vision + audio, on-device, <50ms latency.

2. Reason: Every 10 seconds, the agent asks itself: "Is there something the user needs to know right now?"

3. Whisper: If yes, the glasses speak a 5-10 word prompt in your ear. If no, silence.

The loop runs locally. The cloud is optional. Your data stays on your face.
```

### Q: Is it always listening?
```
Yes, on-device. The mic array is always on, but audio is processed locally and never leaves the device unless you explicitly enable cloud sync (off by default).

You can review, export, or delete every byte of audio at any time. Open source firmware means you can verify this.
```

### Q: What about privacy?

```
We're building a relationship with you, not a data pipeline on you.
```
