## Content Buckets — Recurring Templates

### 1. "The Build Log" (Mondays)
**Format:** 5-7 tweets / LinkedIn post
**Structure:**
- Tweet 1: What we shipped (with screenshot/video)
- Tweet 2: What broke (with stack trace)
- Tweet 3: What we learned
- Tweet 4: What we're shipping next week
- Tweet 5: Ask the community for input

### 2. "The Research Thread" (Tuesdays)
**Format:** 8-12 tweets + GitHub release / arXiv link
**Structure:**
- Tweet 1: The problem (specific, technical)
- Tweet 2: Why existing solutions fail
- Tweet 3: Our approach (one-line summary)
- Tweet 4-8: Architecture / math / diagrams
- Tweet 9: Benchmark results (table)
- Tweet 10: Limitations + future work
- Tweet 11: Code/paper link
- Tweet 12: Ask for feedback

### 3. "The Demo" (Wednesdays)
**Format:** 30-60 sec video + 2-3 tweets
**Structure:**
- Video: Show the proactive AI in action (no narration, just behavior)
- Tweet 1: What you're seeing
- Tweet 2: How it works (one-liner)
- Tweet 3: Link to code

### 4. "The Opinion" (Thursdays)
**Format:** 1-3 tweets / LinkedIn post
**Structure:**
- Hook (controversial or counter-narrative)
- Reasoning (3-5 points)
- What this means for us
- What this means for the industry

### 5. "The Open-Source Drop" (Fridays)
**Format:** GitHub release + announcement thread
**Structure:**
- Repo name + one-liner
- What it does
- Why it matters
- Install command
- Contribute link
