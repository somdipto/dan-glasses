## Feature grid

### 1. Perception
Dan sees your screen and environment through `perceptiond`.

### 2. Audio
Dan hears you through `audiod`, now at v0.6 with adaptive whisper timeout.

### 3. Memory
Dan remembers conversations, preferences, and recurring context through `memoryd`.

### 4. Reasoning
Dan uses a small reasoning model strategy — HRM-Text for the loop, not a giant cloud model for every request.

### 5. Action
Dan can trigger tools, surface reminders, and execute the next useful step.

### 6. Privacy
Data stays local unless you choose to share it.

## Architecture section

### Built as 7 daemons

- `audiod` — voice activity detection + transcription
- `perceptiond` — visual capture and frame scoring
- `memoryd` — episodic, semantic, and procedural memory
- `ttsd` — speech output
- `toold` — tool bridge
- `os-toold` — OS actions
- `openclawd` — orchestration layer

**Supporting line:**
This is not a monolith. It is a system you can inspect, debug, and fork.
